Yes. DevDox uses GitHub App installation tokens which have full access to private repos you grant it access to during installation. Your code never leaves GitHub's infrastructure during fetching.
Only PR metadata is sent — commit messages, PR title, PR description, and linked ticket IDs. Full file contents are not transmitted to any LLM. The analysis runs on commit messages and PR descriptions only.
Currently Groq (Llama 3.1) for fast, free inference. The architecture supports swapping to any LLM — Claude, GPT-4, Gemini — via a single config change. Pro plan will offer model selection.
Every ADR is committed directly to your repository at docs/decisions/YYYY-MM-DD-pr-title-prN.md. It's versioned with your code, searchable, and always readable in plain Markdown.
Currently DevDox triggers on pull_request.opened events only. Draft PR support and re-analysis on PR updates is on the roadmap for the Pro plan.
Go to GitHub Settings → Applications, find DevDox-AI and click Uninstall. All webhook access is immediately revoked. Your existing docs/decisions/ files remain in your repo.
